# 帮我检查.skill

> *发之前多看一眼，很多锅本来可以不背。*

<br>

写了一大段解释，发出去前总觉得不踏实？<br>
怕自己说太多，把底牌、责任和筹码一起送出去了？<br>
怕语气太软，被默认以后还能继续压你？<br>
怕一句话没说稳，被截图、被曲解、被拿来做责任口径？<br>

<br>

**发出去之前，先帮你看一眼这段话到底适不适合发送。**

## 这是什么

`帮我检查.skill` 是一个发送前风险检查器。

它更关注这些问题：

- 这段话会不会暴露底牌
- 会不会揽责过度
- 会不会边界太弱
- 会不会把未确认的信息说死

它输出的不是泛泛的“语气不太好”，而是：

- 最危险的句子
- 为什么危险
- 哪些该删
- 哪些该延后说
- 更稳妥的替换版本

## 支持的数据来源

| 来源 | 聊天记录 | 文档 / PDF | 图片 / 截图 | 代码 / 评论 | 备注 |
|------|:-------:|:----------:|:-----------:|:-----------:|------|
| 待发送文本 | ✅ | — | — | — | 最常见输入 |
| 聊天上下文 | ✅ | — | ✅ | — | 适合判断对方惯常压法 |
| 文档 / PDF / Markdown | — | ✅ | — | ✅ | 适合汇报、事故说明、交接说明 |
| 技术说明 / 代码评论 | ✅ | — | — | ✅ | 适合检查是否暴露不必要细节 |
| 用户补充目标 | ✅ | — | — | — | 例如“不想背锅”“不想太软” |

## 安装

详见 `INSTALL.md`。

如果你只使用最基础的 Skill 能力，当前仓库内容已经够用。
如果你还想补飞书、钉钉、Slack、邮件等材料采集和导出解析，可额外安装：

```bash
pip3 install -r requirements.txt
playwright install chromium
```

## 可选工具

仓库已补充 `tools/` 目录，用于把原始材料整理成更适合 Skill 分析的输入。

详细说明见 `tools/README.md`，第三方来源和许可见 `docs/THIRD_PARTY.md`。

## 使用

在支持 Skill 的运行时中调用：

```text
/check-for-me
```

最常见输入方式：

- 直接贴你准备发出去的话
- 再补一点上下文和对象身份

## 效果示例

**场景一：避免背锅**

```text
用户            ❯ 我准备这样回：这个问题主要是我昨天判断失误……

帮我检查.skill  ❯ 这句话最大的问题是你在结论还没收敛前，先把责任完整兜到自己身上。
                更稳妥的写法：
                目前已确认问题出现在昨天这次变更链路上，我正在补全影响范围和触发条件，晚一点同步完整结论和处理方案。
```

**场景二：防止过度讨好**

```text
用户            ❯ 我想回同事：没事没事我来就好，今晚我加班搞定

帮我检查.skill  ❯ 这条会释放两个信号：
                1. 这件事以后还可以继续压给你
                2. 你的时间成本不需要被考虑
```

**场景三：技术说明收口**

```text
用户            ❯ 这段事故解释是不是说太多了

帮我检查.skill  ❯ 是。当前版本把未经确认的猜测、内部实现细节和责任口径混在了一起。
                建议先拆成：已知事实、当前影响、正在处理、下次同步时间。
```

- 最危险的 1 到 3 句
- 建议保留
- 建议删改
- 改写版本

## 注意事项

- 只输入一段草稿也能用，但补上下文会更准
- 重点是“减少不必要暴露”，不是鼓励不承担责任
- 不帮你撒谎、栽赃、恶意甩锅
